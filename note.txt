Build a complete Arrow Puzzle mobile game in React Native with Expo (using Expo Router, NativeWind for styling, Zustand for state, and React Native Reanimated for animations).



GAME MECHANICS (Read very carefully — this is NOT a rotation game)
Core concept: A grid (e.g. 4x4, 5x5, 6x6) filled with arrows. Each arrow points in one of 4 directions: UP, DOWN, LEFT, RIGHT. The player taps an arrow to make it "escape" the board by flying in the direction it points. An arrow can ONLY escape if its entire path to the edge of the grid is completely CLEAR (no other arrows blocking it). Tapping a blocked arrow does nothing (or shows a shake/error animation). Successfully removing one arrow opens paths for others. The goal is to clear ALL arrows from the board. This is a sequential logic puzzle — order matters.

CORE ENGINE — src/engine/
canEscape(grid: GridState, row: number, col: number): boolean
Check if the arrow at (row, col) has a clear path to the board edge in its pointing direction. Iterate every cell between the arrow and the edge — if ANY cell contains a non-removed arrow, return false. Return true only if the full path is empty.
Example: Arrow at row=2, col=1 pointing RIGHT → check col=2, col=3, col=4... all the way to the right edge. If all empty → can escape.
escapeArrow(grid: GridState, row: number, col: number): GridState
Returns a new grid state with that arrow marked as removed (isRemoved: true, treat as empty cell).
checkWin(grid: GridState): boolean
Return true if removedCount === totalArrows (all arrows cleared).
isDeadlock(grid: GridState): boolean
Return true if NO remaining arrow can escape (all paths blocked) — triggers a "no moves" state.
generateLevel(levelNumber: number): GridState
Use seeded random generation (seed = levelNumber). Generate by REVERSE engineering: start with empty grid, place arrows in a valid removal ORDER (so the puzzle is guaranteed solvable). Grid size increases with level: levels 1-10 → 4x4, 11-30 → 5x5, 31-70 → 6x6, 71-150 → 7x7, 150+ → 8x8. Some cells can be intentionally left empty (dots) for visual spacing.

ANIMATION — Critical for game feel
When an arrow is tapped and CAN escape:

It slides/flies in its direction using withTiming (duration: 300ms) — translate X or Y by 400px (off screen)
It fades out simultaneously with withTiming opacity 0
The cell becomes empty (show a subtle dot like in the original game)
Trigger Haptics.impactAsync(ImpactFeedbackStyle.Light)

When an arrow is tapped but CANNOT escape (blocked):

Play a short shake animation left-right using withSequence
Trigger Haptics.notificationAsync(NotificationFeedbackType.Warning)

When all arrows cleared (WIN):

Show a big celebration animation (Lottie confetti)
Show win overlay with stars, next level button
Trigger Haptics.notificationAsync(NotificationFeedbackType.Success)

LEVEL GENERATION ALGORITHM (seed-based, guaranteed solvable)
1. Start with empty grid of size NxN
2. Create a random removal ORDER using seeded random (seed = levelNumber)
3. For each position in removal order:
   a. Place an arrow pointing toward the nearest empty edge path
   b. The direction must be valid given what's already placed
4. Scramble arrow directions slightly while maintaining solvability
5. Result: a grid where following the correct tap sequence clears all arrows

IMPORTANT RULES FOR THE AGENT

DO NOT use emoji for arrows — use SVG or custom drawn arrow components.
ALWAYS validate solvability — generated levels must be clearable.
ALWAYS use useSharedValue and useAnimatedStyle from Reanimated — never Animated from React Native core.
ALWAYS calculate cell size dynamically from screen width — never hardcode pixel values.
ALWAYS separate pure engine logic (no React) from UI components.
